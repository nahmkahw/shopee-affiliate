# /โพสต์ — Post Social Media Content via API

โพสต์ content จาก `products/{item_id}/content/` ไปยัง platform ที่เลือก

Arguments: $ARGUMENTS
- `YYYY-MM-DD` — โพสต์ทุกสินค้าที่มี post_date ตรงกับวันนั้น
- `{item_id}` — โพสต์เฉพาะสินค้านั้น (ตัวเลข 8+ หลัก)
- `--platform fb,ig,x` — platform ที่จะโพสต์ (ไม่ระบุ = fb,ig,x ทุก platform)

ตัวอย่าง:
```
/โพสต์ 2026-05-18
/โพสต์ 2026-05-18 --platform fb
/โพสต์ 3991346022 --platform fb
/โพสต์ 19283435771 --platform fb,ig
```

## ขั้นตอนที่ 1 — ตรวจสอบ Credentials

อ่านไฟล์ `.env` (ถ้าไม่มีไฟล์ → แจ้งให้ copy จาก `.env.example` แล้วกรอกค่า)

ตรวจว่า platform ที่จะโพสต์มี credentials ครบ:

| Platform | ตัวแปรที่ต้องมี |
|----------|----------------|
| Facebook (`fb`) | `FB_PAGE_ID`, `FB_ACCESS_TOKEN` |
| Instagram (`ig`) | `IG_USER_ID`, `IG_ACCESS_TOKEN`, `IMGBB_API_KEY` |
| X (`x`) | `X_API_KEY`, `X_API_SECRET`, `X_ACCESS_TOKEN`, `X_ACCESS_TOKEN_SECRET` |

ถ้าขาดตัวแปรของ platform ไหน → แสดง **Setup Guide** ของ platform นั้น แล้วหยุดรอให้ผู้ใช้กรอกแล้วแจ้ง "พร้อม"

---

### 🔧 Setup Guide: Facebook

**ต้องมี:** Facebook Page ที่จะโพสต์

1. ไปที่ **developers.facebook.com** → **My Apps** → **Create App**
   - ประเภท: **Business** → ใส่ชื่อ App → Create

2. ใน App Dashboard → **Add Products** → เพิ่ม **Pages API**

3. ไปที่ **Tools** → **Graph API Explorer**
   - เลือก App ที่สร้าง (มุมขวาบน)
   - คลิก **Generate Access Token**
   - เลือก permissions: `pages_manage_posts`, `pages_read_engagement`, `pages_show_list`
   - กด **Generate** → คัดลอก token

4. หา **Page ID**: เปิด Facebook Page → **About** → เลื่อนหา "Page ID"
   หรือรันใน Graph API Explorer: `GET /me/accounts` → ดู `id` ของ Page

5. แปลง token เป็น Long-lived (60 วัน) — รันใน Graph API Explorer:
   ```
   GET /oauth/access_token
     ?grant_type=fb_exchange_token
     &client_id={APP_ID}
     &client_secret={APP_SECRET}
     &fb_exchange_token={SHORT_LIVED_TOKEN}
   ```

6. เพิ่มใน `.env`:
   ```
   FB_PAGE_ID=xxxxxxxxxx
   FB_ACCESS_TOKEN=EAAxxxxxxxx
   ```

---

### 🔧 Setup Guide: Instagram

**ต้องมี:** Instagram Business/Creator account เชื่อมกับ Facebook Page แล้ว + ทำ Facebook setup ก่อน

1. เชื่อม Instagram กับ Facebook Page (ถ้ายังไม่ได้ทำ):
   - ไปที่ Facebook Page → **Settings** → **Linked Accounts** → **Instagram** → **Connect Account**

2. เพิ่ม permissions ใน Graph API Explorer (token เดิม):
   เพิ่ม `instagram_basic`, `instagram_content_publish` แล้ว Generate ใหม่

3. หา Instagram User ID — รันใน Graph API Explorer:
   ```
   GET /{page-id}?fields=instagram_business_account
   ```
   → คัดลอก `id` ใน `instagram_business_account`

4. สมัคร imgBB (ใช้โฮสต์รูปชั่วคราวสำหรับ Instagram API):
   - ไปที่ **imgbb.com** → สมัครฟรี
   - ไปที่ **api.imgbb.com** → **Get API Key** → คัดลอก

5. เพิ่มใน `.env`:
   ```
   IG_USER_ID=xxxxxxxxxx
   IG_ACCESS_TOKEN=EAAxxxxxxxx
   IMGBB_API_KEY=xxxxxxxxxx
   ```

---

### 🔧 Setup Guide: X (Twitter)

**ต้องมี:** X account ที่จะโพสต์

1. ไปที่ **developer.twitter.com** → **Sign in** → **Apply for a developer account**
   - Use case: **Making a bot** หรือ **Building tools for yourself**
   - กรอกคำอธิบาย (ภาษาอังกฤษ ~200 ตัวอักษร):
     *"Personal tool to schedule and post affiliate product reviews on my X account."*

2. เมื่อได้รับอนุมัติ → **Developer Portal** → **Projects & Apps** → **+ New Project** → **+ New App**

3. ใน App settings → **User authentication settings**:
   - เปิด **OAuth 1.0a**
   - **App permissions**: เลือก **Read and Write**
   - Callback URL: `http://localhost`

4. ไปที่ **Keys and Tokens**:
   - คัดลอก **API Key** และ **API Key Secret**
   - กด **Generate** ใต้ **Access Token and Secret** → คัดลอกทั้งคู่

5. เพิ่มใน `.env`:
   ```
   X_API_KEY=xxxxxxxxxx
   X_API_SECRET=xxxxxxxxxx
   X_ACCESS_TOKEN=xxxxxxxxxx
   X_ACCESS_TOKEN_SECRET=xxxxxxxxxx
   ```

## ขั้นตอนที่ 2 — หาสินค้าที่จะโพสต์

รันคำสั่งนี้:

```
node -e "
const fs = require('fs'), path = require('path');
const arg = '$ARGUMENTS'.trim();
const isDate   = /^\d{4}-\d{2}-\d{2}/.test(arg);
const isItemId = /^\d{8,}/.test(arg);
const date   = isDate   ? arg.match(/\d{4}-\d{2}-\d{2}/)[0] : '';
const itemId = isItemId ? arg.match(/\d{8,}/)[0] : '';
if (!date && !itemId) { console.log('ERROR: ระบุวันที่ (YYYY-MM-DD) หรือ item_id ด้วย'); process.exit(1); }
const dirs = fs.readdirSync('products').filter(d => fs.existsSync('products/'+d+'/data.json'));
const items = [];
dirs.forEach(id => {
  const d = JSON.parse(fs.readFileSync('products/'+id+'/data.json','utf8'));
  if (d.status === 'placeholder') return;
  if (date && d.post_date !== date) return;
  if (itemId && d.item_id !== itemId) return;
  const c = path.join('products', id, 'content');
  items.push({ id, title: (d.title||'').substring(0,40), price: d.price,
    fb: fs.existsSync(path.join(c,'facebook.md'))  ? '✓' : '✗',
    ig: fs.existsSync(path.join(c,'instagram.md')) ? '✓' : '✗',
    x:  fs.existsSync(path.join(c,'x.md'))         ? '✓' : '✗' });
});
if (!items.length) { console.log('NONE'); process.exit(); }
items.forEach(p => console.log(p.id+' | '+p.title+' | '+p.price+' | FB:'+p.fb+' IG:'+p.ig+' X:'+p.x));
"
```

แสดงรายการ + platform ที่มี content พร้อมแล้ว แล้วถาม:
**"ยืนยันโพสต์ N รายการ platform [fb,ig,x] ไหม?"**

รอคำตอบก่อนดำเนินการ

## ขั้นตอนที่ 3 — รัน post.js

```
node post.js $ARGUMENTS
```

- timeout 2 นาทีต่อสินค้า
- ถ้า error `token expired` หรือ `OAuthException` → แจ้งให้ต่ออายุ token ตาม Setup Guide แล้วรัน `/post-content` ใหม่
- ถ้า error `rate limit` → แจ้งให้รอ 15 นาทีแล้วลองใหม่
- ถ้า error `ขาด credentials` → ไปขั้นตอนที่ 1 Setup Guide
