# Bubble placement is a fixed corner, not computed from image content

Speech/thought Bubbles are positioned in a fixed corner of their Panel (chosen by the LLM alongside the dialogue text), rather than computed from where the Mascot actually ends up in the generated image. The alternative — detecting the Mascot's position in the Flux Kontext output (e.g. via face detection or saliency) and placing the Bubble in genuinely empty space — was considered and rejected: it would add a CV dependency and a new failure mode (misdetection still risks overlapping the character) for a comic-strip tool where an occasional imperfect overlap is an acceptable cosmetic flaw. Composition prompts hint the model to keep the chosen corner clear, but this is best-effort only. Revisit if overlaps turn out to be frequent enough to hurt the output quality in practice.
